# Data Model: Computer Use Control Plane
**Branch**: `001-computer-use-api`

## Entities

### 1. `Session`
Represents a user's persistent interaction loop context.
- **id**: UUID (Primary Key, autogenerated)
- **user_id**: String (Identifier for the owning user)
- **vnc_environment_id**: UUID (Foreign Key to the isolated target container)
- **state**: Enum (`IDLE`, `EXECUTING`, `ERROR`)
- **created_at**: Timestamp
- **updated_at**: Timestamp

*Validation Rules*: `state` can only transition from IDLE to EXECUTING when an `asyncio.Lock()` is acquired.

### 2. `Message`
Represents an individual event, token stream result, or tool call in the session timeline. Kept in memory during the WebSocket connection and flushed to PostgreSQL upon an `end_turn` stopping reason.
- **id**: UUID (Primary Key)
- **session_id**: UUID (Foreign Key -> Session.id)
- **role**: Enum (`user`, `assistant`, `tool`)
- **type**: Enum (`text`, `thought`, `tool_call`, `tool_result`)
- **content**: JSONB (Dynamic payload containing text, or tool action format dictating mouse/keyboard coordinates, or a base64 encoded screenshot)
- **sequence**: Integer (Guarantees absolute chronological order when restoring history from DB)
- **created_at**: Timestamp

*Validation Rules*: Must map correctly into the Anthropic `messages` API array format upon load.

### 3. `VncEnvironment`
Tracks the dynamic provisioning state of Docker/VNC targets.
- **id**: UUID (Primary Key)
- **host**: String (DNS/IP address for internal routing)
- **port**: Integer (VNC connection port)
- **status**: Enum (`PROVISIONING`, `AVAILABLE`, `IN_USE`, `TERMINATED`)
- **created_at**: Timestamp

*Relationships*: `Session` has a 1:1 relationship with a dynamically provisioned `VncEnvironment` tracking.
